The Mintlify CLI (`mint`) gives you a local preview server with instant hot-reload so you can see your changes before pushing to production. This page covers installation, running the server, useful CLI commands, and common troubleshooting steps.

<Info>
  **Prerequisites**: Node.js v19 or higher and a docs repository containing a `docs.json` file.
</Info>

## Install the CLI

```bash
npm i -g mint
```

Verify the installation:

```bash
mint --version
```

## Start the preview server

Navigate to the root of your docs repository (where `docs.json` lives) and run:

```bash
mint dev
```

The preview is available at [http://localhost:3000](http://localhost:3000). Every time you save a file, the browser refreshes automatically.

## Use a custom port

If port 3000 is already in use, specify a different port with the `--port` flag:

```bash
mint dev --port 3333
```

If the requested port is also taken, the CLI automatically tries the next available port and prints a message:

```
Port 3000 is already in use. Trying 3001 instead.
```

## Validate links

Before pushing, check for broken internal links across your entire docs site:

```bash
mint broken-links
```

The CLI scans every page and reports any links that point to non-existent paths.

## Update the CLI

Each CLI release targets a specific Mintlify version. If your local preview doesn't match production, update to the latest CLI:

```bash
mint update
```

## Deployment

Push to your default branch and the Mintlify GitHub App deploys automatically. A successful deployment shows a green confirmation in your repository's check suite.

<Tip>
  Install the [Mintlify GitHub App](https://dashboard.mintlify.com/settings/organization/github-app) from your dashboard if you haven't already — it's required for automatic deployments.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module'>
    This error usually means your Node.js version is too old. Fix it by:

    1. Remove the current CLI: `npm remove -g mint`
    2. Upgrade to Node.js v19 or higher
    3. Reinstall the CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Page loads as a 404">
    Make sure you're running `mint dev` from the directory that contains your `docs.json` file. The CLI must be run from the docs root.
  </Accordion>

  <Accordion title="Preview doesn't match production">
    Run `mint update` to ensure your local CLI version matches the version your site is built with in production.
  </Accordion>

  <Accordion title="Unknown error on startup">
    Delete the `~/.mintlify` cache folder and restart the CLI:

    ```bash
    rm -rf ~/.mintlify
    mint dev
    ```
  </Accordion>
</AccordionGroup>

For the full CLI changelog, see the [mint package on npm](https://www.npmjs.com/package/mintlify?activeTab=versions).
