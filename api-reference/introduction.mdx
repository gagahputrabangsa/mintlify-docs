The Plant Store API is a sample REST API that demonstrates how Mintlify renders interactive API reference documentation from an [OpenAPI specification](https://mintlify.com/docs/api-playground/openapi/setup). Every endpoint shown here is generated automatically from `api-reference/openapi.json` — no hand-written API docs needed.

<Note>
  This API reference is an example. To build your own API reference, replace `api-reference/openapi.json` with your own OpenAPI spec and update the navigation in `docs.json`.
</Note>

## Authentication

All Plant Store API endpoints require a Bearer token passed in the `Authorization` header:

```bash
curl -H "Authorization: Bearer YOUR_API_TOKEN" \
     https://sandbox.mintlify.com/plants
```

In your OpenAPI spec, authentication is declared as a security scheme:

```json openapi.json
{
  "security": [{ "bearerAuth": [] }],
  "components": {
    "securitySchemes": {
      "bearerAuth": {
        "type": "http",
        "scheme": "bearer"
      }
    }
  }
}
```

Mintlify reads this scheme and adds an **Authorization** field to the interactive API playground automatically — readers can enter their token once and test all endpoints without leaving the docs.

## How Mintlify generates API docs

To generate an API reference like this one from your own OpenAPI spec:

1. Add your spec file to the repo (e.g., `api-reference/openapi.json`)
2. Reference it in `docs.json`:
   ```json docs.json
   {
     "openapi": "api-reference/openapi.json"
   }
   ```
3. Create an MDX file for each endpoint with the `openapi` frontmatter field:
   ```mdx api-reference/endpoint/list-users.mdx
   ---
   title: "List Users"
   openapi: "GET /users"
   ---
   ```

Mintlify renders the full interactive playground, request parameters, response schemas, and code samples automatically.
