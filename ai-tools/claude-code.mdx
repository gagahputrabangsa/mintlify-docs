Claude Code is Anthropic's official command-line AI tool that operates directly in your terminal. Adding a `CLAUDE.md` file to your docs repository teaches Claude about your Mintlify setup, writing conventions, and quality standards — so every Claude session starts with the right context.

## Prerequisites

- An active [Claude subscription](https://claude.ai) (Pro, Max, or API access)
- Node.js installed

## Install Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

Verify the installation:

```bash
claude --version
```

## Create a CLAUDE.md file

Create `CLAUDE.md` at the root of your documentation repository. This file is automatically read by Claude Code at the start of every session:

```markdown CLAUDE.md
# Mintlify documentation project

## Working relationship
- Ask for clarification rather than making assumptions
- Never lie, guess, or invent information
- Push back on ideas when you have a better approach — cite your reasoning

## Project context
- Format: MDX files with YAML frontmatter
- Config: `docs.json` for navigation, theme, and site settings
- Components: Mintlify built-in components (Steps, CodeGroup, Card, etc.)

## Content standards
- Write in second person ("you")
- Document just enough for user success
- Check existing pages for patterns before adding new content
- Avoid duplication unless intentional
- Make content evergreen when possible

## Frontmatter requirements
- `title`: 50-60 characters, keyword-rich
- `sidebarTitle`: 1-3 words for sidebar display
- `description`: 130-155 characters, unique per page

## Writing standards
- Language tags on all code blocks
- Alt text on all images
- Root-relative paths for internal links
- Test all code examples before including them

## Git workflow
- NEVER use --no-verify when committing
- Create a branch for each set of changes
- Commit frequently with clear messages

## Do not
- Skip frontmatter on any MDX file
- Use absolute URLs for internal links
- Include untested code examples
```

## Start a session

Navigate to your docs directory and run:

```bash
cd your-docs-repo
claude
```

Claude Code reads `CLAUDE.md` automatically and is ready to help with documentation tasks.

<AccordionGroup>
  <Accordion title="Write a new page">
    > "Create `guides/authentication.mdx` explaining how to authenticate API requests using Bearer tokens. Include a request example in curl, JavaScript, and Python."
  </Accordion>
  <Accordion title="Edit an existing page">
    > "Update `quickstart.mdx` to include a step for installing the GitHub App. Keep the existing steps and add it as step 3."
  </Accordion>
  <Accordion title="Review a page for quality">
    > "Review `essentials/settings.mdx` and identify any missing frontmatter, broken internal links, or content that could be clearer."
  </Accordion>
  <Accordion title="Generate API docs">
    > "Write an API reference page for the `POST /users` endpoint. The request body needs `name` (string, required) and `email` (string, required). The response returns a user object with `id`, `name`, `email`, and `created_at`."
  </Accordion>
</AccordionGroup>

<Tip>
  Claude Code can read, write, and edit files in your repository directly. Always review changes with `git diff` before committing.
</Tip>
